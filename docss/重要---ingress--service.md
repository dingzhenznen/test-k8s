ingress 背后有个service 在工作 处理请求

在节点中 的hosts 文件中添加了 test-k8s.local

这是一个很好的问题！两者的区别主要在于 Host 头的传递方式：

  ---
  对比
  
  ┌─────────────────────────────────────────────────────┬───────────────────────────────┬────────────────────────────────────────────────┐
  │                        命令                         │            Host 头            │                    工作原理                    │
  ├─────────────────────────────────────────────────────┼───────────────────────────────┼────────────────────────────────────────────────┤
  │ curl http://test-k8s.local                          │ 自动添加 Host: test-k8s.local │ 通过域名解析到 IP，curl 自动把域名放入 Host 头 │
  ├─────────────────────────────────────────────────────┼───────────────────────────────┼────────────────────────────────────────────────┤
  │ curl -H "Host: test-k8s.local" http://10.101.242.21 │ 手动指定 Host: test-k8s.local │ 直接访问 IP，但手动告诉 Nginx 域名             │
  └─────────────────────────────────────────────────────┴───────────────────────────────┴────────────────────────────────────────────────┘

  ---
  详细解释

  方式 1：curl http://test-k8s.local

  ┌─────────────────────────────────────────┐
  │  1. DNS 解析 test-k8s.local             │
  │     ↓                                   │
  │     127.0.0.1（根据 /etc/hosts）         │
  │                                         │
  │  2. curl 连接 127.0.0.1:80              │
  │                                         │
  │  3. HTTP 请求自动包含：                  │
  │     Host: test-k8s.local                │
  │     （curl 自动从 URL 提取域名）          │
  │                                         │
  │  4. Nginx 收到请求，匹配 Host 头         │
  │     ↓                                   │
  │     转发到 test-k8s:8080                │
  └─────────────────────────────────────────┘

  关键点：test-k8s.local 是域名，curl 会自动把它作为 Host 头发送。

  ---
  方式 2：curl -H "Host: test-k8s.local" http://10.101.242.21

  ┌─────────────────────────────────────────┐
  │  1. 直接连接 10.101.242.21:80           │
  │     （跳过 DNS，直接访问 IP）             │
  │                                         │
  │  2. 手动添加 HTTP Header：               │
  │     -H "Host: test-k8s.local"           │
  │                                         │
  │  3. Nginx 收到请求，匹配 Host 头         │
  │     ↓                                   │
  │     转发到 test-k8s:8080                │
  └─────────────────────────────────────────┘

  关键点：直接访问 IP，但手动告诉 Nginx 域名是什么。

  ---
  本质区别
  
  ┌─────────────┬─────────────────┬───────────────────────┐
  │    特性     │ 方式 1（域名）  │ 方式 2（IP + Header） │
  ├─────────────┼─────────────────┼───────────────────────┤
  │ DNS 解析    │ ✅ 需要     │ ❌  │ ❌ 不需要     ─┘       │
  ├─────────────┼─────────────────┼───────────────────────┤
  │ Host 头来源 │ 自动从 URL 提取 │ 手动指定              │
  ├─────────────┼─────────────────┼───────────────────────┤
  │ 灵活性      │ 固定域名        │ 可以访问任意域名      │
  ├─────────────┼─────────────────┼───────────────────────┤
  │ 使用场景    │ 正常访问        │ 测试、调试、绕过 DNS  │
  └─────────────┴─────────────────┴───────────────────────┘

  方式 2：用公共电话 + 报身份
    拨打 10086（公共号码）→ 主动说"我是 XXX" → 客服知道你是谁

    Ingress 的工作原理
  
  ┌─────────────────────────────────────────────────────────┐
  │                    流量流向                              │
  ├─────────────────────────────────────────────────────────┤
  │                                                          │
  │   用户请求                                                │
  │      ↓                                                   │
  │   ┌─────────────────────┐                               │
  │   │  Ingress Controller │  ← 真正处理请求的 Nginx 进程    │
  │   │  (ingress-nginx-    │                               │
  │   │   controller Pod)   │                               │
  │   └──────────┬──────────┘                               │
  │              ↑                                           │
  │   ┌──────────┴──────────┐                               │
  │   │  ingress-nginx-     │  ← Service 暴露 Controller     │
  │   │  controller Service │    让流量能进入                │
  │   │  (10.101.242.21)    │                               │
  │   └──────────┬──────────┘                               │
  │              ↑                                           │
  │   ┌──────────┴──────────┐                               │
  │   │   Ingress 资源       │  ← 定义路由规则（域名→后端）   │
  │   │   (test-k8s-ingress)│                               │
  │   └──────────┬──────────┘                               │
  │              ↓                                           │
  │   ┌─────────────────────┐                               │
  │   │   test-k8s Service  │  ← 后端服务                    │
  │   └──────────┬──────────┘                               │
  │              ↓                                           │
  │   ┌─────────────────────┐                               │
  │   │   test-k8s Pods     │  ← 最终处理请求的应用           │
  │   └─────────────────────┘                               │
  │                                                          │
  └─────────────────────────────────────────────────────────┘

  ---
  各组件的作用
  
  ┌──────────────────────────────────┬────────────────┬───────────────────────────────────────┐
  │               组件               │      类型      │                 作用                  │
  ├──────────────────────────────────┼────────────────┼───────────────────────────────────────┤
  │ Ingress                          │ 资源/配置      │ 定义路由规则（域名→哪个 Service）     │
  ├──────────────────────────────────┼────────────────┼───────────────────────────────────────┤
  │ Ingress Controller               │ Pod/Deployment │ 真正运行的 Nginx 进程，处理 HTTP 请求 │
  ├──────────────────────────────────┼────────────────┼───────────────────────────────────────┤
  │ ingress-nginx-controller Service │ Service        │ 暴露 Controller，让流量能进入集群     │
  └──────────────────────────────────┴────────────────┴───────────────────────────────────────┘

  ---
  关键点
  
  Ingress 资源本身不做任何事情，它只是一份配置：

  apiVersion: networking.k8s.io/v1
  kind: Ingress
  metadata:
    name: test-k8s-ingress
  spec:
    rules:
    - host: test-k8s.local
      http:
        paths:
        - path: /
          backend:
            service:
              name: test-k8s    # ← 这只是配置！
              port:
                number: 8080

  真正工作的是 Ingress Controller：

  1. 监听 Ingress 资源的变化
  2. 读取 Ingress 配置
  3. 生成 Nginx 配置文件
  4. 处理 实际的 HTTP 请求

  │ Ingress 生效靠谁？     │ Ingress Controller（Nginx 进程） │
  ├────────────────────────┼──────────────────────────────────┤

  ├────────────────────────┼──────────────────────────────────┤

  └────────────────────────┴──────────────────────────────────┘

  关系：
  Ingress 资源（配置）+ Ingress Controller（执行）+ Service（入口）= 完整功能

  没有 Ingress Controller，Ingress 资源只是一份"废纸"配置！
